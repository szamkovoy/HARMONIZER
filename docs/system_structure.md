# Регламент структуры проекта Harmonizer

## 1. Глобальная организация
Все функциональные блоки системы располагаются в корневой папке `/modules/`. Каждый модуль является автономным "черным ящиком".

## 2. Внутренняя структура модуля
Каждый модуль (например, `/modules/communicator/`) обязан содержать:
- `ui/` — React-компоненты (интерфейс).
- `core/` — Логика, обработка данных, хуки.
- `api/` — Функции вызова внешних сервисов (Gemini, Hume и т.д.).
- `store/` — Управление локальным состоянием (память сессии).
- `readme.md` — Техническое описание конкретно этого модуля (входы, выходы, текущий статус).

## 3. Документация
- Общие концепции и дорожные карты хранятся в `/docs/`.
- Спецификации конкретных модулей хранятся внутри самих модулей в `readme.md`.
- Курсор обязан обновлять `readme.md` модуля при внесении значительных изменений в его логику.

## 4. Взаимодействие
Модули общаются через четкие интерфейсы (Props). Прямое обращение одного модуля во внутренние папки другого (кроме публичных интерфейсов) запрещено.
