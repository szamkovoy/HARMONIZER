# Дорожная карта Harmonizer (продукт)

## Текущий этап: нативный клиент

- **Выполнено:** переход основного фронтенда с Next.js PWA на **React Native Expo SDK 54** и **Expo Router**; модули в `modules/`, сервисы в `services/`.
- **Сервер:** контракты Gemini и прочие чувствительные ключи остаются на **Vercel** в развёртывании из **`_legacy_web/`** (или выделенного API-слоя с тем же контрактом).
- **Коммуникатор:** голос (`expo-av`), текст, стриминг, задел под эмоции — см. [`modules/communicator_roadmap.md`](modules/communicator_roadmap.md).

## Ближайшие направления

1. **Интеграция Hume** по [`hume_integration.md`](hume_integration.md): выровнять параметры записи (16 kHz, mono, int16 там, где применимо) с пайплайном на нативной стороне.
2. **Модуль ASSISTANT:** роли, шаги диалога, вызов COMMUNICATOR с параметрами режима (`VOICE_ONLY` / `TXT_ONLY` и т.д.).
3. **Стабильность чата:** при росте истории — списки (`FlatList`), оптимизация измерений и скролла.

## Перспектива: биометрия и доверенное устройство

- **Face ID / Touch ID / Android BiometricPrompt** — для входа в приложение, подтверждения чувствительных действий и (опционально) разблокировки сессии с Supabase без постоянного ввода пароля.
- **Secure Store** (например `expo-secure-store`) — хранение токенов и флагов «доверенного устройства».
- Реализация после стабилизации базового auth-потока и политики приватности в продукте.

## Перспектива: сеть и регионы

- Клиент ориентирован на работу через **HTTPS** к развёрнутому API (в т.ч. прокси на Vercel при ограничениях доступа к облачным API из региона).
