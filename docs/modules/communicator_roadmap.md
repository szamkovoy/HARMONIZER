# Дорожная карта развития модуля COMMUNICATOR

## Статус V1 (реализовано)

- Модуль в [`modules/communicator`](../../modules/communicator): UI **VOICE** / **TXT**, стриминг ответа Gemini через HTTP API (**[`_legacy_web/app/api/communicator/route.ts`](../../_legacy_web/app/api/communicator/route.ts)** на Vercel), парсинг `[T]…[/T]`, отмена запроса (`AbortController`), задел `onEmotionSegment`, индикатор «Расшифровка…», автоскролл и подсказка «вниз» при переполнении.
- Клиент: **React Native**, запись — **`expo-av`**, иконки — **`assets/icons/`** (см. [`modules/communicator/readme.md`](../../modules/communicator/readme.md)).
- Демо: экран по умолчанию в приложении — `app/(tabs)/index.tsx` (в архиве веба был маршрут `/communicator`).

## Будущие направления

Для оптимизации работы модуля в нативном приложении предусмотрены следующие пути:

### 1. Двухходовая модель (Flash Speedup)

Если задержка первого токена будет мешать UX:

- **Шаг 1:** лёгкая модель только для быстрой транскрипции.
- **Шаг 2:** параллельно — более «тяжёлая» модель для глубокого ответа.

Реализация на стороне API (`_legacy_web` или отдельный backend), контракт клиента можно сохранить.

### 2. Живое общение (Gemini Live / WebSocket)

Переход с HTTP-стрима на **WebSocket** или Live API:

- меньше задержка, естественный диалог;
- на мобильном — отдельный канал и политика фона (background audio, предупреждения батареи).

### 3. Эмоциональный анализ (Hume AI)

Интеграция по [`../hume_integration.md`](../hume_integration.md):

- вырезание сегмента аудио из записи **expo-av** (длительность и формат по регламенту);
- выравнивание **16 kHz, mono, int16** при подготовке буфера для Hume;
- вызов Hume параллельно или после шага Gemini — см. продуктовые сценарии в ASSISTANT.

### 4. UX нативного чата

- **FlatList** / виртуализация при длинной истории.
- Тонкая настройка скролла к якорю сообщения на разных размерах экрана и с клавиатурой.
