# Дорожная карта развития модуля COMMUNICATOR

## Статус V1 (реализовано)

- Модуль в [`modules/communicator`](/modules/communicator): UI VOICE/TXT, стриминг Gemini 1.5 Flash через [`app/api/communicator/route.ts`](/app/api/communicator/route.ts), парсинг `[T]…[/T]`, отмена запроса, задел `onEmotionSegment`, индикатор «Расшифровка…», автоскролл и кнопка «вниз» при переполнении.
- Демо: маршрут `/communicator`.

## Будущие направления

Для оптимизации работы модуля в будущем предусмотрены следующие альтернативные пути:

## 1. Двухходовая модель (Flash Speedup)
Если задержка первого токена в 1.5 сек будет мешать пользователю, реализовать схему:
- **Шаг 1**: Использование модели **Flash 8B** только для мгновенной транскрипции.
- **Шаг 2**: Параллельный запуск **1.5 Pro** для генерации глубокого психологического ответа.

## 2. Живое общение (Gemini Live)
Переход с REST API на **WebSockets**:
- Обеспечивает мгновенную реакцию без необходимости нажимать/отпускать кнопку.
- Реализация потокового аудио-соединения для естественного диалога.

## 3. Эмоциональный анализ (Hume AI)
Интеграция оценки состояния пользователя:
- Вырезание фрагмента аудио (`audio.slice(0, 15000)`) из первых 15 секунд записи.
- Отправка в Hume API для получения карты эмоций (Prosody) параллельно с основным запросом к Gemini.
