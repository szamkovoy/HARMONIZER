# Технический регламент Hume AI

## Клиент Harmonizer (React Native / Expo)

- Запись пользовательского аудио в приложении выполняется через **`expo-av`** (`Audio.Recording`). Пресеты и контейнеры по умолчанию (например AAC/M4A) могут отличаться от «сырого» PCM; **перед отправкой в Hume** сегмент нужно привести к требованиям ниже (ресэмплинг до **16 kHz**, mono, **int16** — на стороне нативного модуля, backend или утилиты; конкретная реализация фиксируется в задаче интеграции).
- Не хранить и не логировать сырой PCM с ключами в репозитории; буферы обрабатывать в памяти и передавать по HTTPS.

## Параметры аудио для Hume API (критично)

- **Частота:** 16000 Hz.
- **Каналы:** 1 (Mono).
- **Тип данных:** int16 (16-bit PCM).
