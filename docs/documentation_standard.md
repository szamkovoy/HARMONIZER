# Стандарт документации модулей Harmonizer

Все функциональные модули системы должны иметь файл `README.md` в своей корневой папке, а также краткую запись-указатель в `/docs/modules/`.

## Шаблон описания модуля:

1. **Назначение**: Краткое описание цели модуля (какую задачу решает).
2. **Интерфейсы (Inputs/Outputs)**: 
   - Какие параметры принимает (Props, аргументы функций).
   - Что возвращает (события, данные).
3. **Внешние зависимости**: Список используемых API (например, Gemini, Hume) и сторонних библиотек.
4. **Логика работы**: Описание ключевых внутренних функций и алгоритмов.
5. **План развития**: Запланированные улучшения (например, переход на Live API или интеграция новых сервисов).

*Примечание: Документация должна обновляться Курсором при каждом изменении интерфейсов модуля.*
